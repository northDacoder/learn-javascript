The history

In the early days of Twitter, many of their engineers would use different front-end libraries depending on which met their requirements.


Problem

This made scaling and maintanance difficult

Solution

Mark Otto and Jacob created Bootstrap to solve the problem of inconsistency

Commmunity First

After realizing that other developers might find it useful as well, they released it for anyone to use in 2011

Benefits of Bootstrap

  • Easy to start using
  • get started using only html + css
  • DRY
  • Dont Repeat Yourself
  • Stop spending time recreating code that has already been created
  • Consistency
  • Consistent results across different platforms
  • Responsive first
  • Responsive layouts with no additional work
  • Example-based Documentation
  • Built into their documentation/website, you can interact with bootstrap components right next to their code
